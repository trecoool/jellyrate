# JellyRate - Jellyfin User Rating Plugin

## Build & Deploy

```bash
dotnet build -c Release
cp bin/Release/net9.0/JellyRate.dll "//TOWER/appdata/binhex-jellyfin/data/plugins/JellyRate/JellyRate.dll"
```

Jellyfin must be **restarted** after deploying — plugins are loaded at startup only. The plugin folder also needs a `meta.json` (see below).

### meta.json

Jellyfin won't load the plugin without `meta.json` in the plugin folder. If the folder is recreated, this file must be restored. Key fields: `guid` must match `Plugin.cs` (`f3b5e142-7a68-4d5c-9b3a-1e8f6c4d2a09`), `TargetAbi` must match the Jellyfin version.

## Architecture

### Backend (C# / .NET 9.0 / Jellyfin SDK 10.11.0)

- **Storage**: JSON file at `{DataPath}/jellyrate/ratings.json`, loaded into in-memory `Dictionary<Guid, UserRating>` on startup
- **Concurrency**: `SemaphoreSlim` for file writes, `lock` for cache reads
- **API**: `RatingsController` at `/Ratings/*` — all endpoints
- **Script delivery**: The JS is embedded as a resource (`Web/jellyrate.js`) and served via `/Ratings/jellyrate.js`, but the primary injection is via middleware (see below)

### Script Injection (3-layer)

1. **ScriptInjector.cs** (IHostedService) — modifies `index.html` on disk at startup (2s delay)
2. **ScriptInjectionMiddleware.cs** — intercepts HTML responses, injects `<script>` before `</body>`
3. **Controller fallback** — `/Ratings/jellyrate.js` serves the embedded resource

The middleware disables `Accept-Encoding` to prevent gzip, then checks for `text/html` responses to inject into.

### Frontend (jellyrate.js — IIFE, ~700 lines)

The script runs inside Jellyfin's SPA. Key integration patterns:

- **API auth**: Uses `window.ApiClient` for server address and token, constructs `MediaBrowser` auth header manually
- **SPA navigation**: Polls `location.href` every 500ms + listens to `hashchange`/`popstate`
- **Detail page widget**: Retries up to 15x (400ms interval) to find `.detailRibbon` anchor element
- **Card overlays**: `MutationObserver` on `document.body` watches for new `.card` elements, badges them with rating
- **Library tab**: `MutationObserver` watches for `.emby-tabs-slider` appearing, injects "User Rated" tab button

## Jellyfin Web Client Quirks

### Tab System

Jellyfin's library pages use a custom `emby-tabs` Web Component:
- Tab buttons: `button.emby-tab-button` inside `.emby-tabs-slider`
- Content panels: `div.pageTabContent[data-index="N"]`
- Tab buttons are generated by `mainTabsManager.setTabs()` from a `getTabs()` array — we can't hook into this, so we append after rendering
- The `emby-tabs` element fires `beforetabchange` and `tabchange` custom events
- **Critical**: Never add/remove `.hide` on native `.pageTabContent` panels — Jellyfin's tab system manages them internally and can't undo external changes. Instead, use `style.display` and save/restore the original value.
- When the SPA navigates to a library, the entire page DOM is rebuilt — injected tabs are destroyed. The MutationObserver re-injects them.

### URL Patterns

- Library pages: `#!/movies.html?topParentId=xxx`, `#!/tvshows.html?topParentId=xxx`
- Detail pages: `#!/details?id=<guid>`
- The `topParentId` in library URLs is the library's item ID — useful for filtering items by library

### Item Images

Image URLs follow: `/Items/{id}/Images/Primary?maxHeight=450&quality=90`

### DOM Structure (detail pages)

- `.itemDetailPage:not(.hide)` — active detail page
- `.detailRibbon` — action buttons bar (primary anchor for rating widget)
- Media info rows: `.mediaInfoStream` elements for Video/Audio/Subtitles

## Known Issues / Gotchas

- The `margin-left` on `.jellyrate-info-row` pushes the entire row. To add space between the label and content, adjust `.jellyrate-info-label` width or padding instead.
- Stats cache (`allStatsCache`) has a 1-minute TTL — must call `invalidateStatsCache()` after any rating change
- `GetItemById()` can return null for deleted/removed items — always null-check
- Jellyfin card elements use `data-id`, `data-itemid`, or link `href` for item IDs — the code tries all three
- Card IDs may or may not have hyphens — always normalize with `.replace(/-/g, '')` when comparing
